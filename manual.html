工程管理システム Total V1

User Manual - 社内業務効率化のための工程管理ツール

1. システム概要

Total V1 は、国内・海外注文、および製造工程における複雑な進捗をチームでリアルタイムに共有するためのシステムです。Google Apps Script (GAS) をバックエンドに採用し、クラウド上でのデータ一元管理と、オフライン時でも利用可能なJSONバックアップ機能を備えています。

2. ログイン・同期方法

認証の手順

  1. 担当者の選択: プルダウンから自分の名前を選択します。
  2. パスワード入力: 指定されたアクセスパスワードを入力します。
  3. 同期開始: 同期開始ボタンを押すと「ONLINE」状態になり、最新のデータが読み込まれます。

※認証に成功すると、3分ごとに自動的に最新の状態へ更新（オートシンク）されます。

3. 案件の登録

新しい案件（プロジェクト）を追加する際の手順です。

  • 案件NO: 社内の管理番号を入力します。
  • 案件名: 識別しやすい名称を入力します。
  • テンプレート機能:
    • 国内注文：請求、入金、製造、発送の4工程を自動生成
    • 海外注文：Iv/Pl作成、通関手続き等を含む6工程を生成
    • 製造工程：原料手配から品質検査までの5工程を生成
  • 優先度設定: 至急 🔥 / 重要 / 通常 から選択します。

4. 工程（タスク）の詳細操作

各案件のヘッダーをクリックすると、詳細な工程テーブルが開きます。

機能 説明
重要度 (★) 工程ごとの重要度を5段階で設定。星が多いほど重要です。
期限設定 カレンダーから選択。期限が切れると赤く、3日以内になると黄色く強調されます。
担当者割当 工程ごとに担当者を個別設定できます。
ステータス 「未着手」「進行中」「完了」を切り替えます。完了すると行がグレーアウトします。

5. 便利な表示・検索機能

自分宛フィルタ

自分宛チェックボックスをオンにすると、自分が担当者に設定されている「未完了の工程」がある案件のみが表示されます。朝のタスク確認に最適です。

案件の並べ替え

  • 優先度順: 至急案件を最上部に表示します。
  • 期限が近い順: 全工程の中で最も期限が迫っている案件から順に並べます。

6. データのバックアップとログ

JSON保存と読込

万が一ネットワークが不安定な場合や、過去の特定の状態を保存しておきたい場合は、JSON保存を利用してください。保存したファイルは読込からいつでも復元可能です。

システムログ

画面最下部の黒いボックスには、誰がいつどの操作（案件追加、状態変更など）を行ったかがリアルタイムで記録されます。ミスの追跡や作業履歴の確認に役立ちます。
